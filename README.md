# GameBot — 自动化清体力脚本

基于图像识别 + 状态机的游戏日常副本自动化脚本。支持多角色轮换、自定义连招、兜底连招、隐身反检测。提供完整的图形界面，零编程基础即可使用。

---

# 第一部分：快速开始

## 环境要求

- Windows 10/11
- Python 3.10+
- **管理员权限运行**（必须，否则无法向游戏发送输入）

## 安装

```bash
pip install -r requirements.txt
```

## 启动

**方式一：源码运行**

双击 `start.bat`（自动以管理员权限启动），或在管理员终端中运行：

```bash
python gui.py
```

**方式二：exe 运行**

直接双击 `GameBot.exe` 即可启动（exe 内嵌管理员权限，系统会自动弹出 UAC 确认窗口）。

> exe 打包方法见文档底部「打包为 exe」章节。

## 首次配置步骤

### 1. 设置游戏窗口

- 将游戏切换为**窗口化模式**运行（不支持全屏独占模式）
- 在 GUI「预设管理」页面的「窗口标题」中填写游戏窗口标题关键词（如"晶核"）

### 2. 截取模板图片

使用 GUI「截图工具」页面截取以下模板图片，保存到 `templates/` 目录。建议为每个角色创建子文件夹（如 `templates/镰卫/`）。

**全局模板**（所有角色共用）：

| 模板 | 用途 | 截取建议 |
|------|------|---------|
| 进入游戏按钮 | 选人界面的"进入游戏"按钮 | 截取按钮完整区域，包含文字 |
| 再次挑战按钮 | 副本结束后的"再次挑战"按钮 | 截取按钮完整区域 |
| 退出副本按钮 | 副本结束后的"退出副本"按钮 | 截取按钮完整区域 |
| 副本出口图标 | 副本通关后地面的出口光圈标记 | 截取图标本体，不要包含过多背景 |
| NPC寻路图标 | NPC头顶的可交互图标 | 截取图标本体 |
| 设置按钮 | 城镇中按ESC后出现的设置齿轮图标 | 截取图标本体 |
| 切换角色按钮 | 设置菜单中的切换角色按钮 | 截取按钮区域 |
| 确认按钮 | 各种确认弹窗中的确认按钮 | 截取按钮区域 |

**角色模板**（每个角色独立）：

| 模板 | 用途 | 截取建议 |
|------|------|---------|
| 选人头像 | 选人界面中该角色的头像 | 截取头像框内区域，约 70x70 像素 |
| 技能栏 | 战斗界面右下角的技能图标之一 | 截取单个技能图标，约 55x55 像素 |
| 结算画面 | 战斗结束后结算面板上该角色的特征区域 | 约 80x80 像素 |
| 城镇头像 | 城镇界面左上角的角色头像 | 约 65x55 像素 |

### 3. 配置预设

在「预设管理」页面：
1. 填写全局模板路径（步骤2截取的图片文件名）
2. 配置城镇导航操作链（从城镇到副本入口需要点击的按钮序列）
3. 添加角色到角色列表，为每个角色配置刷本次数和连招

### 4. 录制或配置连招

在「连招管理」页面：
- **录制**：按 F5 开始录制，在游戏中执行一轮技能操作，再按 F5 停止，保存
- **手动**：点击「新建」，逐个添加按键动作

### 5. 启动运行

1. 在「运行控制」页面选择预设
2. 设置执行角色数
3. 点击「启动」

## 注意事项

- **必须以管理员权限运行**，否则无法向游戏发送鼠标和键盘输入
- 游戏窗口必须为**窗口化模式**，不支持全屏独占
- 运行期间不要移动游戏窗口，不要在游戏窗口上方覆盖其他窗口
- 刷深渊时角色一开始不要待在莱茵城，黑街的嘉尔姆NPC无法寻路
- 首次使用建议先用 1 个角色测试完整流程，确认无误后再增加角色数

# 第二部分：功能详细介绍

## 状态机工作流

Bot 运行时按以下流程自动执行：

```
选择角色 → 城镇导航 → NPC寻路 → 副本加载 → 副本战斗
   ↑                                            ↓
   │                                   副本出口寻路
   │                                   ├── 再次挑战 → 副本加载（循环）
   │                                   └── 退出副本 → 地图加载
城镇退出 ←────────────────────────────────────┘
   ├── 切换角色 → 选择角色（循环）
   └── 全部完成 → 停止
```

## GUI 页面功能

### 运行控制

| 控件 | 说明 |
|------|------|
| 预设选择 | 下拉选择已保存的预设 |
| 执行角色数 | 从预设角色列表中执行前 N 个角色 |
| 从第几个角色开始 | 跳过前面的角色，从指定位置开始 |
| 隐身模式 | 启用类人行为：鼠标贝塞尔曲线移动、随机延迟抖动 |
| 后台模式 | 启用虚拟显示器，游戏在虚拟屏幕运行，释放主屏幕（需安装VDD驱动） |
| 完成后退出 | 全部角色完成后是否退出游戏 |
| 启动/停止 | 启动或停止 Bot |
| 日志面板 | 实时显示运行状态，支持 INFO/DEBUG 级别切换 |

### 预设管理

预设分为以下配置区域：

#### 进入游戏

| 配置项 | 说明 |
|--------|------|
| 进入游戏按钮 | 选人界面"进入游戏"按钮的模板图片 |
| 窗口标题 | 游戏窗口标题的关键词，用于自动查找游戏窗口 |

#### 城镇导航

| 配置项 | 说明 |
|--------|------|
| 操作链 | 从城镇到进入副本需要依次点击的按钮模板列表（如：日常按钮 → 副本选择 → 副本入口） |
| 确认进入链 | 进入副本前的确认弹窗按钮列表 |
| NPC图标 | NPC头顶的可交互图标模板（用于自动寻路到NPC） |
| Alt呼出鼠标 | 是否需要按Alt键呼出鼠标光标（部分游戏需要） |

#### 副本战斗

| 配置项 | 说明 |
|--------|------|
| 副本出口图标 | 通关后地面出口光圈的模板 |
| 再次挑战按钮 | "再次挑战"按钮模板 |
| 退出副本按钮 | "退出副本"按钮模板 |
| 确认按钮 | 确认弹窗按钮模板 |

#### 退出流程

| 配置项 | 说明 |
|--------|------|
| 设置按钮 | ESC后出现的设置图标模板 |
| 切换角色按钮 | 设置菜单中的切换角色按钮模板 |
| 退出游戏按钮 | 退出游戏按钮模板 |
| 确认退出按钮 | 退出确认弹窗按钮模板 |

#### 角色列表

每个角色可配置：
- **刷本次数**：该角色刷几次副本
- **连招**：绑定的连招文件
- **兜底连招**（预设级别）：自定义连招执行完毕后的循环连招

### 角色库管理

角色库独立于预设存储在 `config/characters/` 目录，一个角色一个 JSON 文件。每个角色有 4 个模板：

| 模板 | 作用 | 识别场景 | 推荐尺寸 |
|------|------|---------|---------|
| 选人头像 | 在选人界面识别并点击该角色 | 选择角色阶段 | 70x70 |
| 技能栏 | 确认已进入副本战斗 | 副本加载/NPC寻路阶段 | 55x55 |
| 结算画面 | 识别战斗结束 | 副本战斗阶段 | 80x80 |
| 城镇头像 | 确认已回到城镇 | 地图加载/副本退出阶段 | 65x55 |

### 连招管理

#### 录制连招
1. 切换到「连招管理」页面
2. 按 F5 开始录制（3秒倒计时后开始）
3. 在游戏中执行一轮技能操作
4. 再按 F5 停止录制
5. 查看录制结果，点击「保存」

录制的第一个技能前延会自动提升到至少 1 秒，避免过早释放。

#### 手动配置连招
点击「新建」创建空连招，逐个添加按键动作：
- **按键**：支持 1-5、q、e、w、a、s、d、space、left_click、right_click 等
- **时长**：按键持续时间（秒）
- **前延/后延**：动作前后的等待时间（秒）
- **按住**：是否持续按住不放
- **重复**：重复执行次数

### 截图工具

1. 点击「全屏截图」捕获当前屏幕
2. 在截图上拖选区域
3. 输入文件名，保存到 `templates/` 目录

### 开发者工具

| 功能 | 说明 |
|------|------|
| 起始状态选择 | 从任意状态开始运行（跳过前面的状态） |
| 模板批量匹配测试 | 截取当前屏幕，遍历所有角色模板进行匹配测试 |

## 模板配置技巧

### 如何截取高质量模板

1. **只截取目标区域**：不要包含过多背景，背景像素会降低匹配精度
2. **避免半透明区域**：按钮边缘的半透明渐变区域会导致匹配波动
3. **尺寸适中**：
   - 太小（<40px）：特征不足，容易误匹配
   - 太大（>150px）：背景干扰多，跨分辨率适配差
   - 推荐：50-100px 区间
4. **每个角色单独截取**：不同角色的技能栏/结算画面不同，必须分别截取

### 置信度调节

每个模板都可以配置置信度阈值（默认 0.65）：

| 阈值 | 效果 |
|------|------|
| 0.50-0.60 | 宽松匹配，误匹配风险高 |
| 0.65-0.75 | 推荐范围，平衡精度与召回 |
| 0.80-0.90 | 严格匹配，可能漏匹配 |

**调节建议**：
- 使用「开发者工具」中的「模板批量匹配测试」查看各模板的实际置信度
- 如果运行中某模板频繁匹配失败，适当降低阈值
- 如果出现误匹配（匹配到错误位置），适当提高阈值

### 减少误匹配

| 方法 | 适用场景 |
|------|---------|
| 提高置信度 | 模板有唯一性但阈值偏低 |
| 启用颜色校验 (`color_threshold`) | 大面积彩色模板（如角色头像） |
| 启用翻转拒绝 (`reject_flip`) | 模板左右不对称（如UI按钮、技能图标） |
| 重新截图 | 模板边缘模糊或包含过多背景 |

颜色校验和翻转拒绝可在角色库编辑对话框中配置，或直接编辑角色 JSON 文件。

## 卡死恢复机制

当画面持续 15 秒无变化时，自动进入卡死恢复：
1. 释放所有按键
2. 反复按 ESC 并识别设置按钮（最多 3 轮 x 4 次）
3. 识别到设置按钮 → 进入正常退出流程
4. 全部失败 → 停止 Bot

如果运行期间反复触发卡死恢复（累计 3 次），Bot 会自动停止。

## 后台模式（实验性）

基于虚拟显示器驱动(VDD)，将游戏移到虚拟屏幕运行，释放主屏幕供日常使用：
- 首次使用会自动提示安装 VDD 驱动
- 启动时自动将游戏移到虚拟显示器
- 停止时自动将游戏移回主屏幕
- 运行期间主屏幕可正常使用其他应用

## 命令行用法

```bash
python gui.py                           # GUI 模式（推荐）
python main.py --list                   # 列出所有预设
python main.py -p 深渊 -c 3             # 用"深渊"预设刷 3 个角色
python main.py -p 深渊 --stealth        # 隐身模式
python main.py --record-combo 我的连招   # 录制连招
python build.py                         # 打包为 exe
```

## 打包为 exe

确保已安装 PyInstaller：

```bash
pip install pyinstaller
```

执行打包：

```bash
python build.py
```

打包完成后，在 `dist/` 目录下生成 `GameBot.exe`。exe 特性：

- **单文件**：所有代码和依赖打包为一个 exe
- **内嵌管理员权限**：双击自动弹出 UAC 确认，无需手动提权
- **无控制台窗口**：运行时不会弹出黑色命令行窗口
- **不包含用户数据**：模板、预设、角色库、连招文件不会打包进 exe

### 运行 exe

将 `GameBot.exe` 放入一个独立文件夹，首次运行时程序会自动创建以下目录结构：

```
GameBot/
├── GameBot.exe
├── templates/          # 放入截取的模板图片
├── config/
│   ├── presets/        # 放入预设 JSON
│   └── characters/     # 放入角色库 JSON
└── combos/             # 放入连招 JSON
```

### 分发给其他用户

将以下文件一起打包分发：

```
GameBot.exe
templates/              # 你的模板图片
config/presets/          # 你的预设文件
config/characters/       # 你的角色库文件
combos/                  # 你的连招文件
```
